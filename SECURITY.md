# Security Policy

We take the security of **[APP_NAME]** seriously. Thank you for helping keep the
app and its users safe.

## Supported Versions

Only the latest released version receives security updates. Please make sure you
are running the most recent release before reporting an issue.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately through either of the following channels:

1. **GitHub private vulnerability reporting (preferred)** — open the repository's
   **Security** tab and click **Report a vulnerability**. This keeps the report
   confidential and lets us collaborate on a fix directly.
2. **Email** — send details to **focalstudio.apps@gmail.com**.

When reporting, please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof-of-concept
- Affected version(s), device, and OS
- Any suggested mitigation, if known

## What to Expect

- We will acknowledge your report within **5 business days**.
- We will investigate and keep you informed of our progress.
- Once a fix is released, we will credit you in the release notes if you wish.

Please give us a reasonable amount of time to address the issue before any
public disclosure. We appreciate responsible disclosure and your help in
protecting our users.
